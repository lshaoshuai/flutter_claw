# 🦀 Flutter Claw (AI Agent Sandbox OS)

**Flutter Claw** is a lightweight AI Agent operating system architecture designed specifically for
mobile devices. It enables Large Language Models (LLMs) to interact directly with mobile hardware
via generated JavaScript (ES6) code, running within a local, physically isolated **QuickJS Sandbox
**.

---

## ✨ Core Features

* 🧠 **Dual-Brain Drive**: Native adaptation for Gemini 2.0 Flash (v1beta API), featuring high
  instruction following and ultra-low latency.
* 🛡️ **Isolated Execution**: All AI-generated code runs within ClawJSRuntime (QuickJS), ensuring the
  security of the main process.
* 📳 **Emotional Feedback System**: Built-in emotion perception logic. When the Agent feels "happy,"
  it autonomously triggers physical haptic feedback.
* 🔗 **Highly Extensible Bridge**: Easily expose native functions (clipboard, sensors, network,
  database) to the AI via ClawBridgePlugin.
* 🔄 **Self-Healing Loop**: Features code extraction validation and automatic retry mechanisms for
  error handling, effectively managing JS syntax errors.

---

## 🛠️ System Architecture

| Layer                    | Component      | Responsibility                                                                 |
|:-------------------------|:---------------|:-------------------------------------------------------------------------------|
| **Perception (Manager)** | ManagerAgent   | Processes System Prompts, extracts JS code, and manages self-correction loops. |
| **Brain (LLM)**          | GeminiProvider | Encapsulates Gemini 2.0 protocols for logic reasoning and code generation.     |
| **Sandbox (Runtime)**    | ClawJSRuntime  | Execution environment based on flutter_js; manages JS lifecycle and callbacks. |
| **Bridge (Registry)**    | BridgeRegistry | Unified management of native plugins under sys, network, vfs namespaces.       |

## 🚀 Quick Start

### 1. Configure Android Permissions

```
Add the vibration permission in android/app/src/main/AndroidManifest.xml:

<uses-permission android:name="android.permission.VIBRATE"/>
```

### 2. Core Bridge Implementation Example

Define a system plugin on the Dart side to expose capabilities:

```dart
class SystemPlugin extends ClawBridgePlugin {
  @override
  String get namespace => 'sys';

  @override
  Map<String, dynamic Function(List<dynamic>)> get methods =>
      {
        'vibrate': (args) {
          HapticFeedback.heavyImpact(); // Triggers physical haptic feedback
          return '{"status": "ok"}';
        },
      };
}

```

### 3. Agent Instruction Flow

When a user praises the Agent, the AI perceives the emotion and automatically generates/executes the
following JS:

```javascript
// Automatically generated by AI upon perceiving positive emotion
Claw.sys_vibrate();
Claw.finish("Oh stop it! You're making my heart race (literally)!");
```

## 🏗️ Execution Workflow

1. User Input: Send text or multimodal data to the ManagerAgent.
2. Code Generation: LLM reasons and outputs a response containing an executable code block.
3. Sandbox Injection: Code is extracted and injected into the ClawJSRuntime.
4. Native Call: JS calls mapped native methods through the global Claw object.
5. Result Return: The Agent decides to reply to the user or trigger a Self-Healing Loop based on the
   execution status.
